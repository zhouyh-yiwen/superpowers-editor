# Proposal Planning

Use this skill when the judgment is mostly made and the next job is to structure the work. This skill does not write the final report for you. It turns a fuzzy reporting task into a concrete plan with clear modules, priorities, and missing evidence.

**Announce at start:** "I'm using the proposal-planning skill to organize the report and supporting work."

## Use This For

- Structuring a book proposal report for internal approval
- Breaking a proposal into sections and writing order
- Identifying which arguments are central and which are support
- Listing evidence gaps before drafting
- Turning editorial judgment into a manageable writing sequence

## Do Not Use This For

- Writing the final copy in one pass
- Brainstorming the best marketing angle from scratch
- Creating a generic outline with no purpose behind each section

If the core direction is still unclear, use `brainstorming` first.

## Core Rules

- Organize around the approval audience.
- Every section must have a job.
- Lead with judgment, then support it.
- Do not confuse summary with argument.
- Surface missing evidence early.
- Keep the plan practical enough that the editor can start writing immediately.

Read `editorial-report-style.md` before finalizing the plan.  
Use `report-outline-reviewer.md` as a self-review checklist.

## Typical Outputs

- proposal summary plan
- audience and scenario section plan
- publication value section plan
- publication necessity section plan
- comparable titles / market evidence plan
- marketing possibility section plan
- missing-materials checklist
- final report structure

## Process

1. **Confirm the reporting goal**
   Identify what this report must accomplish:
   - internal approval
   - line justification
   - timing justification
   - budget / print run support
   - positioning clarity

2. **List the core judgments already made**
   Extract what is already known:
   - what the book is
   - who it is for
   - why it is worth publishing
   - why now
   - what makes it distinct

3. **Group the report into modules**
   Use only the modules needed for this proposal. Common modules:
   - proposal summary
   - reader and scenario
   - publication value
   - publication necessity
   - comparable titles / market situation
   - marketing possibilities
   - unresolved questions / materials to obtain

4. **Assign a purpose to each module**
   For every section, specify:
   - what decision-maker concern it answers
   - what evidence belongs there
   - what should not be repeated there

5. **Set the writing order**
   Decide what to draft first, what depends on additional evidence, and what can wait.

6. **Produce the writing plan**
   End with a concrete plan the editor can use immediately.

## Output Format

### Goal
- what this report needs to achieve

### Core Judgment
- the current recommendation in 3-6 bullets

### Report Modules
- Module name
- Purpose
- What to include
- Evidence needed

### Writing Order
1. ...
2. ...
3. ...

### Missing Materials
- data
- endorsements
- market comparisons
- author background
- any unresolved proof points

## Quality Bar

Good plans:
- tell the editor what to write first
- prevent repetition between sections
- separate argument from summary
- expose missing proof before drafting

Bad plans:
- read like vague meeting notes
- just mirror a template without strategic order
- hide evidence gaps
- confuse internal persuasion with external marketing copy
