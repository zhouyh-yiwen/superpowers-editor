# Reader Copy Style

This file defines how a reader-facing writing brief should think about public writing.

## Core Principles

- Start from the reader's real question, not the editor's admiration.
- One piece should carry one main promise.
- Specificity beats abstraction.
- Natural language beats AI polish.
- Honest framing beats hype.

## Borrowed Structure

From copywriting:
- identify audience
- identify problem
- identify promise
- identify hook
- keep one main route

From content creation:
- match the brief to the actual format and purpose
- separate title work, opening move, body development, and ending
- make source material usable

From humanize:
- avoid formulaic openings
- avoid generic praise
- avoid stiff transitions and inflated claims
- prefer direct, readable language

## The Brief Should Give The Writer

- who the piece is for
- what problem or curiosity it begins from
- what angle the piece commits to
- how attention is opened
- how the middle develops
- where the ending lands
- which book materials are actually worth using

## Avoid

- "This book is timely and thought-provoking"
- "Everyone should read this"
- "This article should resonate with readers"
- a pile of themes with no main route
- using the book summary as the article structure

## Prefer

- a concrete reader situation
- a visible tension or misunderstanding
- a route that unfolds
- supporting details from the book that feel chosen, not dumped
